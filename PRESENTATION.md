# Surge Report — 발표용 PPT 산출물

> **한 줄 프로젝트 정의**
> 엔드투엔드 자동화로 **급등주·저평가 주식을 실시간 감지·분석·알림**하는 개인 투자자 대시보드 플랫폼

---

## 1. 문제 정의 (Problem Statement)

| 기존 Pain Point | 영향 |
|---|---|
| 급등주는 **장중 5~15분 내** 사라지는 기회 | 놓치면 수익 창출 불가 |
| 1,000+ 종목 수동 모니터링 불가 | 개인 투자자의 시간·정신적 부담 |
| 전문가 인사이트가 흩어져 있음 (YouTube, 네이버, DART 등) | 정보 과잉 + 신뢰도 판단 어려움 |
| 국민연금·투자고수 매매는 공시 후 수 일 지나서야 확인 | 시점 지연, 벤치마킹 효과 감소 |

**핵심 문제**
> 개인 투자자는 **실시간으로 "왜 급등했는지" + "누가 샀는지" + "언제 들어갈지"**를 한 화면에서 판단할 도구가 없다.

[PPT 변환 포인트]
- **슬라이드 제목**: "왜 개인 투자자는 급등주를 매번 놓치는가?"
- **시각자료**: 😫 → ⏱️ → 💸 흐름 아이콘 3단계
- **도식화**: 기회 윈도우 타임라인 (장중 09:00~15:30, 강조 구간 09:10~09:40)
- **강조 메시지**: "급등주는 기회가 아니라 **시간과의 싸움**이다"

## 2. 기존 방식의 한계 (Before)

### 2.1 정보 수집 방식 비교

| 항목 | 기존 (수동) | 한계 |
|---|---|---|
| 급등주 발굴 | 네이버 증권 → 거래대금 TOP 클릭 | **실시간 아님**, 과거데이터 노출 |
| 전문가 의견 | YouTube 20분 영상 시청 | **시간 소모**, 핵심 포인트 추출 어려움 |
| 국민연금 매매 | DART 공시 사이트 검색 | **3개월 단위 보고**, 실시간성 제로 |
| 알림 | 카카오톡 단톡방, 유료 리딩방 | **정보 과잉**, 신뢰도 불명 |
| 저평가 주식 | 재무제표 수기 분석 | **전문성 필요**, 시간 지연 |

## 4. 핵심 기능 (Core Features)

### 4.1 기능 매트릭스

| 기능 | 설명 | 기술적 의의 | 사용자 가치 |
|---|---|---|---|
| **급등주 실시간 감지** | 상승률 + 거래량 급증 + 시가총액 필터 | API 호출 → 즉시 판별 | 기회윈도우 5~15분 확보 |
| **5등급 시스템 (S~D)** | S=초급등(8%+), A=강세(5%+), B/C/D=하락포함 | 룰 기반 + 재현성 | 우선순위 명확화 |
| **저평가 주식 발굴** | PER/PBR/배당률 + 전문가 인사이트 융합 | 다중 지표 융합 | 숨은 가치 종목 선별 |
| **전문가 인사이트 큐레이션** | 오일전문가 + 세상학개론 자동 추출 | NLP 키워드 기반 요약 | 20분 영상 → 3줄 핵심 |
| **국민연금 대량보유 추적** | DART API → NPS 보유 공시 실시간 수집 | 공시 데이터 자동화 | 기관 수급 방향성 확인 |
| **섹터 히트맵** | 급등주 섹터 집계 → 색상 밀도 시각화 | 실시간 집계 엔진 | 시장 기류 한눈에 |
| **Telegram 알림** | S/A등급 → 개인 4명 + 그룹챗 동시 발송 | 멀티채널 푸시 | 놓치지 않는 알림 |
| **네이버 토론방 연동** | 종목 클릭 → 실시간 토론방 이동 | 딥링크 통합 | 커뮤니티 심리 확인 |

### 4.2 등급 판별 로직 (룰 엔진)

```
┌─────────────────────────────────────────┐
│           급등주 판별 로직              │
├─────────────────────────────────────────┤
│  입력: 등락률, 거래량, 시가총액, PER   │
├─────────────────────────────────────────┤
│  IF 등락률 ≥ 8%  → S-Grade (👑 초급등) │
│  IF 등락률 ≥ 5%  → A-Grade (🔥 강세)   │
│  IF 등락률 ≥ 3%  → B-Grade (📈 상승)   │
│  IF 등락률 ≥ 0%  → C-Grade (➡️ 보합)   │
│  IF 등락률 < 0%  → D-Grade (📉 하락)   │
├─────────────────────────────────────────┤
│  + 저평가 필터: PER < 10 AND PBR < 1    │
│  → 저평가 주식 목록 별도 생성           │
└─────────────────────────────────────────┘
```

## 7. 기대 효과 (Expected Impact)

### 7.1 정량적 효과

| 지표 | Before | After (예상) | 개선율 |
|---|---|---|---|
| 급등주 발견 시간 | 30~60분 후 | **장중 5분 내** | **90% 단축** |
| 정보 수집 채널 | 5~8개 사이트 | **1개 대시보드** | **80% 통합** |
| 전문가 인사이트 소요 | 20분/인물 | **3줄 요약/즉시** | **95% 시간 절약** |
| 국민연금 공시 확인 | 수동 검색 (주 1회) | **자동 수집 (일 1회)** | **자동화 100%** |
| 알림 누락률 | 수동 확인 의존 | **Push 100% 발송** | **0% 누락** |

### 7.2 정성적 효과

- **정보 불안감 ↓**: 모든 데이터가 한 화면에 집결
- **FOMO 감소 ↓**: 중복 알림 방지 + 캐시 시스템으로 과잉 알림 차단
- **학습 효과 ↑**: 히트맵 + 등급 분포로 시장 흐름 체득
- **신뢰도 ↑**: DART 공시 + 전문가 인사이트 → 검증 가능한 정보원

[PPT 변환 포인트]
- **슬라이드 제목**: "90% 빠르게, 80% 적게, 100% 놓치지 않게"
- **시각자료**: Before/After 비교 막대그래프 4개
- **도식화**: 개선율을 % 원형 차트로 표현
- **강조 메시지**: "**시간은 줄이고, 신뢰는 높인다**"

## 8. 기술적 차별점 (Technical Differentiation)

### 8.1 기술 스택 선정 이유

| 기술 | 선정 이유 | 대안 vs 선택 |
|---|---|---|
